# Fraud Transport Dashboard

Единый Streamlit-проект для поиска мошенничества в общественном транспорте:

- rule-based гипотезы находят известные и объяснимые признаки фрода;
- ML-инференс использует готовую discovery-модель и ищет кандидатов на новые паттерны.

## Куда положить модель

Готовый файл модели должен лежать здесь:

```text
artifacts/ml/discovery_model.pkl
```

Если модель обновилась, просто замените этот файл и заново запустите ML-инференс в Streamlit.
Обучение модели в основном сценарии не запускается.

## Основные папки

```text
data/raw/              исходные CSV/XLSX/ZIP транзакций
artifacts/rules/       результаты rule-based гипотез
artifacts/ml/          features, labels, ML-инференс и discovery_model.pkl
src/rules/             движок правил
src/ml/score.py        инференс готовой ML-модели
src/services/pipeline.py orchestration
src/ui/                стили и UI-компоненты
```

## Запуск

```bash
streamlit run app.py
```

## Загрузка данных через интерфейс

На первом экране Streamlit есть блок `Загрузите транзакции`.
Можно выбрать один или несколько CSV-файлов и запустить:

- `Анализировать правилами` — только rule-based гипотезы;
- `Правила + ML` — гипотезы, построение признаков и инференс готовой модели.

Загруженные файлы сохраняются во временной папке текущей сессии. Папка `data/raw/`
при этом не изменяется.
